⏺ IBM Granite TimeSeries TTM R2 — 下載與使用指南

  安裝

  pip install granite-tsfm

  ---
  快速開始：零樣本預測 (Zero-Shot)

  from tsfm_public.toolkit.get_model import get_model

  # 自動選擇最適合的模型變體
  model = get_model(
      model_path="ibm-granite/granite-timeseries-ttm-r2",
      context_length=512,       # 歷史資料長度
      prediction_length=96,     # 預測長度
  )

  get_model() 會根據你指定的 context_length 和 prediction_length 自動從
  HuggingFace 下載對應的模型分支。

  ---
  直接用 Transformers 載入

  from transformers import AutoModel

  model = AutoModel.from_pretrained(
      "ibm-granite/granite-timeseries-ttm-r2",
      revision="512-96-r2"  # 指定模型變體分支名稱
  )

  ---
  微調 (Fine-Tuning) 範例

  from tsfm_public.models.tinytimemixer import TinyTimeMixerForRegression
  from torch.optim import Adam

  model = TinyTimeMixerForRegression.from_pretrained(
      "ibm-granite/granite-timeseries-ttm-r2"
  )

  optimizer = Adam(model.parameters(), lr=0.001)
  for epoch in range(num_epochs):
      predictions = model(input_data=scaled_features)
      loss = criterion(predictions, targets)
      loss.backward()
      optimizer.step()

  ---
  模型變體命名規則

  格式：<context_length>-<prediction_length>-<freq_ft>-<loss>-<release>

  ┌──────────────────┬─────────┬────────────┬────────────────────────┐
  │     變體範例     │ context │ prediction │          說明          │
  ├──────────────────┼─────────┼────────────┼────────────────────────┤
  │ 512-96-r2        │ 512     │ 96         │ 基本款                 │
  ├──────────────────┼─────────┼────────────┼────────────────────────┤
  │ 512-96-ft-mae-r2 │ 512     │ 96         │ 帶頻率調整 + MAE loss  │
  ├──────────────────┼─────────┼────────────┼────────────────────────┤
  │ 1024-192-r2.1    │ 1024    │ 192        │ r2.1 版，支援日/週頻率 │
  ├──────────────────┼─────────┼────────────┼────────────────────────┤
  │ 1536-96-r2       │ 1536    │ 96         │ 更長上下文             │
  └──────────────────┴─────────┴────────────┴────────────────────────┘

  ---
  重要注意事項

  1. 必須對資料做標準化 — 每個 channel 獨立做 standard scaling，這是強制要求
  2. 不要用 zero-padding 湊 context length — 會嚴重影響效能
  3. Rolling forecast 不建議超過 2× prediction_length
  4. 模型極小 (~805K 參數) — 可在 CPU 或筆電上運行，不需要 GPU
  5. r2 支援 分鐘/小時級頻率；r2.1 額外支援 日/週級頻率

⏺ 這是實驗 #11（context=1024, prediction=48）的預測結果圖。

  圖表解讀

  兩條線：
  - 黑色線（Actual）— 驗證集中 1KTI09 燒成帶溫度
  的實際值，幾乎是一條水平線，穩定在 799.41°C 附近
  - 紅色線（Predicted）— 模型預測值，在 799.39~799.45°C 之間波動

  Y 軸範圍： 799.40 ~ 799.45°C，僅約 0.05°C
  的範圍。這個刻度非常放大，所以紅線看起來波動劇烈，但實際上偏差極小。

  MAE = 0.0134°C 表示模型預測平均只偏離實際溫度 0.013 度。

  為什麼看起來「差很多」但其實很好

  這張圖的 Y 軸被極度放大了。如果把 Y 軸拉到完整的溫度範圍（比如
  798~802°C），兩條線會幾乎完全重疊。紅線的波動本質上是模型在「猜測」一個幾乎不
  變的信號的微小變化，這些波動都在 ±0.05°C 以內。

  值得注意的現象

  黑色實際值幾乎完全平坦（799.41°C），說明在這段驗證時段內，窯溫非常穩定。模型的
  紅色預測線則展現出一些「虛假波動」—
  這是模型根據歷史模式預期的溫度變化，但實際並未發生。這在穩態工況下是正常的：模
  型學到了「溫度通常會有小幅波動」的模式，即使真實數據恰好很平穩。
